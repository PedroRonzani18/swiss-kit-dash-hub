# Security Policy

## Reportando vulnerabilidades

Se você identificar uma vulnerabilidade de segurança, não abra uma issue pública.

Envie o relato por canal privado para o mantenedor do repositório (via contato do perfil GitHub de `@PedroRonzani18`), incluindo:

- descrição do problema
- impacto potencial
- passos para reprodução
- evidências técnicas (logs, payloads, prints)

## Processo de resposta

- O mantenedor fará uma triagem inicial e confirmará o recebimento.
- Se a vulnerabilidade for válida, será tratada com prioridade.
- A divulgação pública deve ocorrer somente após correção e alinhamento.

Este processo é uma política inicial e pode ser atualizado com um contato dedicado no futuro.
